# 에이전트 사용 가이드: 기존 vs 새로운 3-Agent 시스템

## 현재 상황

### 기존 에이전트
- **이름**: "Diff report explainability specification"
- **용도**: 일반적인 작업 (분석, 수정, 구현 등 모든 작업)
- **특징**: 범용 에이전트, 모든 작업을 처리 가능

### 새로운 3-Agent 시스템
- **Agent 1**: 요구사항 분석 전문
- **Agent 2**: 코드 설계 전문
- **Agent 3**: 구현 및 테스트 전문
- **특징**: 역할 분리, 체계적인 워크플로우

---

## 언제 어떤 에이전트를 사용할까?

### 기존 에이전트 사용 (권장)

**사용 시나리오:**
- ✅ 일반적인 작업 (대부분의 경우)
- ✅ 빠른 수정이 필요한 경우
- ✅ 간단한 질문이나 확인
- ✅ 기존처럼 작업하고 싶을 때
- ✅ 복잡도가 낮은 작업

**예시:**
```
"Diff report explainability specification" 에이전트에게:
- "이 코드가 뭐하는 거야?"
- "버그 수정해줘"
- "이 함수 리팩토링해줘"
- "문서 작성해줘"
```

**장점:**
- ✅ 빠르고 간단
- ✅ 모든 작업을 한 번에 처리
- ✅ 익숙한 방식

---

### 3-Agent 시스템 사용

**사용 시나리오:**
- ✅ 복잡한 기능 개발
- ✅ 체계적인 설계가 필요한 작업
- ✅ 대규모 리팩토링
- ✅ 문서화가 중요한 프로젝트
- ✅ 여러 단계가 필요한 작업

**예시:**
```
복잡한 작업:
1. Agent 1: 요구사항 분석
2. Agent 2: 설계 문서 작성
3. Agent 3: 구현 및 테스트
```

**장점:**
- ✅ 체계적인 접근
- ✅ 높은 품질 보장
- ✅ 문서화 자동화

---

## 실전 사용 전략

### 전략 1: 기본은 기존 에이전트 사용

**대부분의 작업:**
```
"Diff report explainability specification" 에이전트에게:
- 일반적인 작업 요청
- 빠른 수정
- 간단한 질문
```

**복잡한 작업만 3-Agent 사용:**
```
- 새로운 기능 개발
- 대규모 리팩토링
- 아키텍처 변경
```

---

### 전략 2: 작업 복잡도에 따라 선택

**간단한 작업:**
```
→ 기존 에이전트 사용
또는
→ Agent 3 직접 사용
```

**복잡한 작업:**
```
→ 3-Agent 시스템 사용 (Agent 1 → 2 → 3)
```

---

### 전략 3: 하이브리드 사용

**일반 작업:**
```
기존 에이전트: "Diff report explainability specification"
```

**특정 작업:**
```
- 분석이 필요할 때 → Agent 1
- 설계가 필요할 때 → Agent 2
- 구현이 필요할 때 → Agent 3
```

---

## 비교표

| 기준 | 기존 에이전트 | 3-Agent 시스템 |
|------|--------------|----------------|
| **용도** | 모든 작업 | 복잡한 작업 |
| **속도** | 빠름 | 느림 (3단계) |
| **품질** | 좋음 | 매우 좋음 |
| **문서화** | 수동 | 자동 |
| **사용 빈도** | 높음 (대부분) | 낮음 (복잡한 작업만) |
| **학습 곡선** | 낮음 | 중간 |
