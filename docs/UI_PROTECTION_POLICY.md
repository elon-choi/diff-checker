# UI 보호 정책

## 목적
AI 어시스턴트가 사용자 명시적 요청 없이 UI를 변경하거나 삭제하는 것을 방지하기 위한 정책입니다.

## 핵심 원칙

### 1. UI 변경 금지 원칙
- ❌ **절대 금지**: 사용자가 명시적으로 요청하지 않은 UI 변경/삭제
- ✅ **필수**: 모든 UI 변경은 사용자 확인 후에만 수행
- ✅ **예외 없음**: "개선"이나 "최적화"를 이유로 기존 UI 변경 금지

### 2. UI 보호 주석 규칙
모든 UI 컴포넌트에는 다음 주석을 추가합니다:

```typescript
// ⚠️ UI PROTECTED: 이 섹션은 사용자 명시적 요청 없이 변경/삭제 금지
// 변경 시 반드시 사용자 확인 필요
// 마지막 검증: YYYY-MM-DD
```

### 3. UI 변경 프로세스
UI를 변경해야 할 때는 반드시 다음 절차를 따릅니다:

1. **변경 의도 명시**: 사용자에게 어떤 UI를 어떻게 변경할지 명확히 설명
2. **사용자 확인**: 사용자로부터 명시적 승인 받기
3. **변경 전 스냅샷**: 변경 전 상태 문서화
4. **변경 후 검증**: 변경 후 사용자에게 확인 요청

### 4. 보호 대상 UI 컴포넌트
다음 UI 컴포넌트는 특히 보호가 필요합니다:

- 헤더 영역 (Phase 선택, Run Diff 버튼, Export 버튼)
- Input 영역 (Spec, Figma, Web, Android, iOS 입력)
- Summary 영역
- Findings 영역
- Next 섹션

### 5. 위반 시 조치
- 즉시 원복
- 변경 사항 롤백
- 사용자에게 사과 및 재발 방지 대책 수립

## 구현 방법

### 방법 1: 보호 주석 추가
모든 UI 컴포넌트에 보호 주석을 추가하여 AI가 변경 시 경고를 받도록 합니다.

### 방법 2: UI 스냅샷 테스트
UI 변경 시 자동으로 스냅샷 테스트를 실행하여 회귀를 감지합니다.

### 방법 3: UI 컴포넌트 분리
UI 컴포넌트를 별도 파일로 분리하여 변경 추적을 용이하게 합니다.

### 방법 4: 변경 감지 알림
UI 관련 파일 변경 시 자동으로 사용자에게 알림을 보냅니다.

## 권장 사항

가장 효과적인 방법은 **방법 1 (보호 주석) + 방법 3 (컴포넌트 분리)** 조합입니다:
- 보호 주석으로 AI가 변경을 인지
- 컴포넌트 분리로 변경 추적 용이
- 명확한 책임 분리
